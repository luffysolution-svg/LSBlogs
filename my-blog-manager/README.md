# 本地博客管理器

这是博客模板配套的本地图形管理器。它只在当前电脑上运行，不需要在线管理后台。

## 推荐目录结构

```text
你的工作目录/
├─ LSBlogs/            # 正式博客和 Git 仓库
├─ my-blog-manager/    # 本地管理器
└─ Start-Blog.bat      # 可选的博客启动脚本
```

管理器会自动寻找同级的 `LSBlogs`。如果使用了其他目录名，可在“设置 > 项目仓库设置”中选择一次博客目录。该绝对路径保存在 `manager_data/deploy_config.json`，已被 Git 忽略，不会进入模板仓库。

也可以在启动管理器前设置环境变量 `LSBLOGS_BLOG_PATH`，让管理器自动识别自定义位置。

`LSBlogs` 是已发布内容的唯一数据源。管理器会直接读取和修改其中的文章、杂谈、瞬间、相册、友链、项目和站点配置；`my-blog-manager` 只保存未发布草稿、待处理队列及本机私密设置。旧版管理器遗留的独有内容可在“管理台”中显式迁移，迁移不会覆盖正式博客内已有的同名文件。

## 日常发布

1. 使用编辑器修改文章、杂谈、瞬间或站点设置。
2. 打开右上角收件箱，点击“保存到本地”。
3. 点击“检查并发布”，核对目录、远程仓库、分支和文件清单。
4. 确认后，管理器会对正式博客执行生产构建，提交新增、修改和删除的相关文件，再推送 `origin`。
5. 如果 Vercel 已连接该分支，推送完成后会自动部署。

发布器不会执行 `git add .`，不会生成或修改 SSH 密钥。GitHub 登录凭据继续使用电脑上已有的 Git 配置。

## 图床

“设置 > 多云图床”支持 Lsky Pro、腾讯云 COS、阿里云 OSS 和 GitHub Contents API。密钥保存在被 Git 忽略的 `manager_data/picbed_config.json`，上传时不会发送站点配置中的凭据。云存储 Bucket 或 GitHub 仓库需要允许公开读取图片，建议给密钥配置最小的 Bucket/仓库权限。
